PROJECT BRIEF — Document Validation API (100% Own Implementation)
Objetivo general:
Crear una API REST profesional usando Node.js y Express.js que valide documentos (imágenes/PDF) completamente de forma local, sin usar servicios externos. La API debe recibir uno o varios archivos, procesarlos, ejecutar múltiples validaciones técnicas y devolver un JSON estructurado con el resultado. El proyecto debe seguir buenas prácticas profesionales, estar bien organizado, incluir seguridad, documentación y ser digno de portafolio/GitHub.
Objetivos específicos:
– Recibir documentos a través de multipart/form-data
– Validarlos: formato, MIME, tamaño, resolución, histograma, si están en blanco, si están corruptos, si cumplen calidad mínima
– Generar metadatos: hash SHA256, resolución, tamaño, tipo MIME
– Devolver un JSON limpio con valid=true/false, errores, metadatos
– Añadir seguridad: helmet, rate limiting, CORS, sanitización, límites de tamaño de archivo
– Arquitectura profesional de carpetas
– Documentación completa en README (o Swagger opcional)
– Código listo para interviews / GitHub / portfolio

Validaciones que debe implementar la API:

Validaciones básicas:


Que file_1 exista siempre
MIME permitido (image/jpeg, image/png, application/pdf)
Comprobar tamaño máximo (ej. 5MB)
Comprobar tamaño mínimo (ej. 50KB)
No permitir archivos vacíos


Validaciones avanzadas (con Sharp):


Leer resolución width/height
Comprobar resolución mínima (ej. 600px)
Detectar imágenes completamente blancas o negras (histograma simple)
Detectar corrupción de archivo (Sharp lanza error)
Extraer metadata básica
Detectar rotación EXIF y corregirla (opcional)


Validaciones opcionales:


Contraste mínimo
Brillo mínimo
Proporciones razonables
Comprobar si el documento ocupa la mayor parte de la imagen
Validar si hay contenido variado en la imagen (no un lienzo plano)


Validación PDF (opcional):


Tipo MIME real (no fiarse de extensión)
Número de páginas
Tamaño
Peso


Hash SHA-256:


Calcular el hash del archivo
Incluirlo en la respuesta
Poder almacenar hashes para evitar duplicados (opcional)

Endpoint principal:
POST /api/validate-document
multipart/form-data
fields:

file_1: required
file_2: optional

Respuesta ejemplo (OK):
{
"valid": true,
"errors": [],
"metadata": {
"mime": "image/jpeg",
"sizeKB": 450,
"width": 1920,
"height": 1080,
"hash": "ab318d..."
}
}
Respuesta ejemplo (KO):
{
"valid": false,
"errors": [
"mime_not_allowed",
"resolution_too_low",
"file_too_small"
]
}
Seguridad obligatoria:

helmet()
CORS con whitelist (localhost y dominio futuro)
rate limiter: 100 req / 15min
limitar tamaño máximo de body (ej. 10MB)
sanitización de inputs
borrar archivos temporales después de validar
tratar errores con un middleware centralizado

Manejo de errores:

error.middleware.js gestiona todos los errores
devolver errores tipificados (mime_not_allowed, corrupted_file…)
middleware 404
